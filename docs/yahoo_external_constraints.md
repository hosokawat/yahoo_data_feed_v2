# Yahoo外部仕様サマリー（本プロジェクト反映版）

確認日: 2026-02-28  
対象: Yahoo広告 動的ディスプレイ広告の商品リスト関連仕様

## 1. 商品リストファイル仕様

- 1行目はヘッダー、2行目以降は商品データ。
- 対応文字コード: `UTF-8` / `UTF-16` / `Shift_JIS`
- 対応改行コード: `LF` / `CRLF`
- 対応形式: `CSV` / `TSV` / `ZIP`
- サイズ上限:
  - CSV/TSV: 150MB
  - ZIP: 60MB
- 商品件数上限: 300,000件
- アップロード回数上限: 1日あたり全件更新4回、差分更新4回
- 失敗アップロードも回数として消費される。

## 2. フィード項目・更新方式の制約

- フィード項目は全29項目。
- 並び替えは可能だが、ヘッダー名は正規名称のみ有効。
- 値がなくてもヘッダー列は削除不可。
- 同一商品リスト内で `Item ID` 重複不可。
- 必須項目: `Item ID`、`Item Name`、`Description`、`Landing Page URL`
- `Delete` 項目は差分更新専用。
- 全件更新で `Delete` 項目が存在するとエラーになる。

## 3. URL・画像関連制約

- `Landing Page URL` / `Smartphone Landing Page URL` / `Tracking URL` / `Image URL` は1024文字以内。
- URLはRFC3986非準拠を許容しない（日本語URL等は不可）。
- URL先頭が `.` で始まる値は不可。
- 画像は事前ホスティングが必要。
- 画像更新時は同一URL差し替えではなく、`Image URL` 文字列自体の変更が必要。

## 4. 価格表示関連制約

- 価格表示優先順位:
  - `Formatted Sale Price`
  - `Sale Price`
  - `Formatted Price`
  - `Price`
- `Price` / `Sale Price` を使用する場合、広告設定のプレフィックス/サフィックス設定が必要（または自動補完表示あり）。

## 5. 通常アップロード運用制約

- 実運用手順は「エラーチェック」実施後にアップロード実行。
- 更新失敗商品は更新前データで配信継続される。
- 失敗時のエラーZIP確認が運用上必須。
- 審査は通常約5営業日目安（混雑・再アップロードで延長可能性あり）。

## 6. 定期更新（FTP/SFTP）制約

- 定期更新は `ftp://` または `sftp://` での取得。
- 絶対パス必須、末尾拡張子は `.csv` / `.tsv` / `.zip`
- URL埋め込みでの認証情報指定は不可（専用項目で設定）。
- 更新頻度は時間単位・日次・週次の設定。
- 定期更新クローラーの公開IPレンジ: `203.216.252.128/26`

## 7. 広告入稿側の前提制約（本アプリ外）

以下は商品リストファイル出力外だが、実運用で満たす必要がある。

- ボタン文言は入稿必須（管理画面プルダウン）。
- 主体者表記は入稿必須（20文字以内）。
- ロゴ画像は指定3サイズが必須（400x200、600x80、600x600）。
- キャンペーンバナーは任意。

## 8. 配信前提（運用要件）

- 商品閲覧履歴活用配信には、計測タグ/サイトリターゲティングタグ/アプリ連携のいずれかが必要。
- 商品リストファイル作成とアップロードは必須プロセス。

## 9. 本プロジェクトへの適用方針

- 本アプリは商品リストファイル生成までを責務範囲とする。
- 本アプリはローカルTSV出力に加えて、任意のSFTPサーバーへの手動アップロード機能を提供する。
- Yahoo管理画面での入稿、審査、運用更新は責務範囲外とする。
- 初版（v1）はYahoo管理画面への手動アップロード運用とする。
- ただし、出力ファイルはYahoo受け入れ制約に適合することを必須要件とする。
- 初版（v1）は全件更新専用運用とし、差分更新（`Delete` 運用）は将来対応とする。
- URL検証は実務上の簡易チェックを採用し、厳密RFC3986判定は将来対応とする。
- 初版（v1）の出力固定値は `TSV` / `UTF-8` / `LF` とする。
- 将来拡張として、SFTP経由の定期更新運用（スケジュール実行・公開鍵認証）に対応可能な設計を維持する。
