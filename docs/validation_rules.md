# 入力・出力バリデーション仕様

確認日: 2026-02-28  
適用対象: Yahoo商品リスト出力機能

## 1. 検証タイミング

- 入力チェック実行時: 全件検証（Error/Warning 一覧更新）
- ローカル出力実行時: 全件検証で出力可否判定
- SFTPアップロード実行時: 全件検証でアップロード可否判定

## 2. 出力中断エラー（Error）

| ID | ルール | 対象 | 条件 |
|---|---|---|---|
| E-001 | 必須ヘッダー未設定 | 1行目 | 必須4項目のいずれかがヘッダーに存在しない |
| E-002 | ヘッダー名不正 | 1行目 | 正規29項目に存在しない名称を含む |
| E-003 | 必須項目空欄 | データ行 | 必須4項目のいずれかが空欄 |
| E-004 | 商品ID重複 | データ行 | `Item ID` が同一値で複数行存在 |
| E-005 | URL長超過 | データ行 | URL系項目が1024文字超過 |
| E-006 | URL形式不正 | データ行 | 簡易URLチェックに不合格（`http://` or `https://` で開始、空白なし、先頭`.`禁止） |
| E-007 | 件数上限超過 | 全体 | データ行件数が300,000件超過 |

## 3. 警告（Warning）

| ID | ルール | 条件 |
|---|---|---|
| W-001 | ファイルサイズ警告 | 出力見込みサイズが150MB超過見込み |
| W-002 | 未使用列警告 | 複数列が全行空欄（ヘッダーのみ） |
| W-003 | 価格項目混在警告 | `Price/Sale Price` と `Formatted*` を同時利用 |
| W-005 | Delete列除外通知 | `Delete` 列または値を検出したため、全件更新用として出力時に除外 |

## 4. 更新方式ルール

### 4.1 全件更新（v1対応）

- `Delete` ヘッダーを出力しない（入力されていても除外）。
- 全商品を置換対象として扱う。

### 4.2 差分更新（将来対応）

- v1では未対応とする。

## 5. エラー表示要件

- エラーは `種別 / 行 / 列 / メッセージ` で表示する。
- 同一セルに複数エラーがある場合、一覧表示する。
- 出力実行時にErrorが1件以上あればファイルを生成しない。

## 6. URL簡易チェック仕様（v1）

- 対象項目: `Tracking URL`、`Landing Page URL`、`Smartphone Landing Page URL`、`Image URL`
- 空欄は許容（必須項目の必須性チェックは別ルールで実施）。
- 値がある場合のみ以下を満たすこと。
  - `http://` または `https://` で開始する。
  - 文字列中に半角/全角スペースを含まない。
  - 先頭文字が `.` ではない。
  - 1024文字以内（`E-005`）。
- 厳密RFC3986準拠判定はv1では実施しない。
