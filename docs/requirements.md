# Yahoo広告データ変換アプリ 要件定義書

## 1. 目的
Google Spreadsheet からコピーした表データを Electron アプリ上のスプレッドシート風 UI に貼り付け、Yahoo 広告向けフォーマットで出力できるようにする。

## 2. スコープ
- 対象
  - デスクトップアプリ（Electron）
  - 商品データの貼り付け、編集、マッピング、出力
- 非対象
  - Google Spreadsheet API 連携（認証含む）
  - Yahoo 広告 API への直接アップロード
  - Yahoo管理画面への自動アップロード（v1）

## 3. 技術スタック
- Electron
- TypeScript
- React
- Tailwind CSS

## 4. 機能要件
### FR-01: 表データの貼り付け
- ユーザーは Google Spreadsheet 上の複数セルをコピーし、アプリにペーストできること。
- タブ区切り・改行区切りの表データをセル構造として取り込むこと。

### FR-02: スプレッドシート風編集
- 表示は行列グリッドで行い、セル単位で編集できること。
- 2行目以降を商品データ行として扱うこと。

### FR-03: 1行目マッピング行
- 1行目は「Yahoo のどの項目にマッピングするか」を選択する専用行とすること。
- 各列の 1 行目セルはプルダウン選択であること。
- 初期値は必須4項目（`Item ID`、`Item Name`、`Description`、`Landing Page URL`）のみを設定すること。
- 任意項目はユーザーが列追加してマッピングできること。
- 必須項目の未設定や重複を検出し、出力前にエラー表示すること。
- ヘッダー名は Yahoo の正規名称のみ選択可能とし、`"` で囲んだ文字列は無効とすること。
- 初版（v1）では全件更新専用運用とし、`Delete` 項目は出力対象外として扱うこと。

### FR-04: 出力先フォルダ指定
- ユーザーは出力先フォルダをダイアログで選択できること。
- 最終選択フォルダをアプリ設定に保存し、次回起動時に復元すること。

### FR-05: ファイル出力
- マッピング行（1行目）をヘッダーとして、2行目以降をデータ行として出力すること。
- 出力時にバリデーションを実施し、エラーがある場合は出力を中断すること。
- 出力成功時は保存先パスとファイル名を表示すること。
- v1では、生成したファイルをユーザーがYahoo管理画面へ手動アップロードする運用とすること。

### FR-06: 入力チェック
- ユーザーは「入力チェック」ボタンで、出力前に全件バリデーションを手動実行できること。
- チェック結果は Error / Warning として一覧表示すること。

### FR-07: SFTPアップロード
- 「データ出力機能」内に `ローカル出力` と `SFTPアップロード` のタブを持つこと。
- SFTPタブでは `host` / `port` / `username` / `password` / `remoteDirectory` を入力できること。
- 実行時はローカル出力と同一のバリデーションを実施し、Error がある場合はアップロードを中断すること。

### FR-08: 設定保存
- 以下を設定ファイルへ保存し、次回起動時に復元すること。
  - 出力先フォルダ
  - ヘッダー構成
  - SFTP設定（`password` は `savePassword=true` の場合のみ保存）
  - UI状態（冒頭説明メッセージ表示、出力機能パネル開閉、参考資料パネル開閉）
  - ウィンドウ位置・サイズ

### FR-09: Yahooアップロード前提バリデーション
- 商品ID（`Item ID`）はデータ行内で重複不可とし、重複行をエラーにすること。
- 必須項目（`Item ID`、`Item Name`、`Description`、`Landing Page URL`）の空欄行をエラーにすること。
- URL項目（`Tracking URL`、`Landing Page URL`、`Smartphone Landing Page URL`、`Image URL`）は文字数上限と実務上の簡易形式チェックを検証すること（厳密RFC3986判定は行わない）。
- 出力は全件更新用のみ対応すること。
- `Delete` 項目は出力時に常に除外すること（入力されていても削除指示としては扱わない）。

## 5. UX要件
- Google Spreadsheet からのコピペを阻害しない操作感を優先すること。
- 行・列を視認しやすいスプレッドシート風 UI とすること。
- 1行目（マッピング行）はデータ行と視覚的に区別すること（色・固定表示など）。
- エラーはセル位置（行/列）を特定して表示すること。

## 6. 出力要件
### 6.1 出力フォーマット
- 区切り: TSV（タブ区切り）※固定
- 文字コード: UTF-8 ※固定
- 改行コード: LF ※固定
- 互換要件: Yahoo 側受け入れ仕様（UTF-8/UTF-16/Shift_JIS, LF/CRLF）に対して UTF-8/LF で互換を満たすこと。

### 6.2 命名規則
- 仕様に沿った標準命名規則を以下で確定する。
- `yahoo_feed_{YYYYMMDD}_{HHmmss}_{recordCount}rows.tsv`

命名要素:
- `{YYYYMMDD}`: 出力日（例: 20260228）
- `{HHmmss}`: 出力時刻（例: 093015）
- `{recordCount}`: データ行件数（2行目以降の件数）

ファイル名例:
- `yahoo_feed_20260228_093015_120rows.tsv`

### 6.3 同名ファイル衝突時
- 同名ファイルが存在する場合、末尾に連番 `_v2`, `_v3` を付与して保存すること。

### 6.4 Yahoo仕様に基づく出力制約
- 1行目はヘッダー行、2行目以降を商品行とすること。
- 未使用項目はヘッダーを残し、2行目以降を空欄で出力できること。
- 1ファイルの商品件数上限は 300,000 件とし、超過時は出力前にエラーとすること。
- TSV出力時のファイルサイズは 150MB を超える見込みの場合に警告を出すこと。

## 7. データ仕様
- グリッド列数は初期状態で必須4項目列（`Item ID`、`Item Name`、`Description`、`Landing Page URL`）のみとする。
- 標準 29 項目の定義（項目名・必須/任意・重複可否）は [yahoo_feed_fields.md](./yahoo_feed_fields.md) で管理し、アプリ起動時に読み込むこと。
- 1行目の選択肢は標準 29 項目のみとする（初版要件）。
- Yahoo仕様上、`Delete` は差分更新用途の特別項目であるため、初版（全件更新専用）では非採用とすること。

## 8. 画面要件
- メイン画面
  - 貼り付け可能なグリッド
  - データ出力機能パネル（折りたたみ可）
  - ローカル出力タブ（出力先フォルダ選択、TSV出力）
  - SFTPアップロードタブ（接続情報入力、アップロード実行）
  - 入力チェックボタン
  - Yahoo参考資料パネル（折りたたみ可）
  - バリデーション結果表示エリア

## 9. 非機能要件
- 対応 OS: macOS / Windows（初版）
- 10,000 行程度の貼り付けで UI 操作が継続可能であること（著しいフリーズがないこと）。
- 主要操作（貼り付け、セル編集、出力）でアプリが異常終了しないこと。

## 10. 受け入れ基準
- Google Spreadsheet からコピーした 29 列以上のデータを貼り付けできる。
- 1行目がマッピング選択行として動作し、初期値が必須4項目のみで表示される。
- 2行目以降の編集が可能で、編集内容が出力ファイルに反映される。
- 出力先フォルダを指定してファイル出力できる。
- 出力ファイル名が命名規則に一致する。
- バリデーションエラー時に出力が止まり、エラー内容が表示される。
- 必須4項目未入力、商品ID重複、URL不正を検出できる。
- `Delete` 項目が入力されていても、全件更新用出力では除外してファイル生成できる。
- SFTPタブで同一バリデーション後にアップロードできる。
- パネル折りたたみ状態と説明メッセージ表示状態が次回起動時に復元される。
